C++Primer01
Zhang 作者
  2023-05-08 19:56:11 2023-05-08 19:56   2023-05-08 19:58:16  
  • c++ 
 
  • c++primer 

一种数据类型不仅定义了数据元素的内容，还定义了这类数据上可以进行的运算。

在执行一个程序时，Windows上可以忽略其扩展名，linux上不可以。

我们可以通过编译器的参数设置，来进行编译行为的选择，GNU编译器上使用-wall，微软编译器使用/W4

流（stream）表达的是随着时间的推移，字符是顺序生成和消耗的。

iostream 包含 istream 和 ostream 两个基础类型。

标准库一共定义了四个基础的IO对象 cin cout cerr clog

<< 运算符接受两个运算对象，左侧的运算对象必须是一个ostream对象，右侧的运算对象则是要打印的值。

>> 运算符接受两个运算对象，左侧的运算对象必须是一个istream对象，右侧的运算对象则是要打印的值。

标准库定义了不同版本的输入输出运算符来处理这些不同的类型的运算对象。

文件重定向可以使我们直接从文件导向文件。

endl是一个被称为操纵符的特殊值，效果是结束当行，并将与设备关联的缓冲区中的内容刷到设备中。缓冲刷新可以保证到目前为止的程序所产生的所有输出都真正写到输出流中，而不是仅停留在内存中等待写入流。

::域命名运算符前缀可以指定域命名空间，命名空间可以帮助哦我们避免不经意间的名字定义冲突，以及使用库中相同的名字导致的冲突，标准库所有的名字都在命名空间std中。

注释可以分为单行注释和界定符对注释，当界定符对注释跨越多行时，我们采用注释内每一行都以一个星号开头，从而指出整个范围都是多行注释的一部分。

界定符对注释不可以嵌套，在调试时需要注释掉一些代码，最好使用单行注释来避免嵌套错误。

多种控制流语句可以让我们写出不同的执行路径，选择语句可以让程序执行条件执行。

当我们使用一个istream对象作为条件时，效果是检测流的状态，如果流是有效的，那么检测成功。如果遇到文件结束符，或无效输入，那么检测失败。

编译器的检查是检查形式上的错误，一般有，语法错误，类型错误（命名空间和标识符拼写），声明错误。

错误信息经常是一个行号和一句简述，由于单个错误具有传递效应，我们最好在每修正一个错误后就立即重新编译代码。（编辑-编译-调试）

c++的程序风格有多种，我们一旦选择了风格就应该坚持下来。

我们通过定义一个类来定义自己的数据结构，类定义了一个类型，以及与其关联的一组数据操作。

如果使用一个类，我们不必关系它是如何实现的，只需要知道类对象可以执行什么操作。

点运算符只能用于类类型的对象，其左侧运算对象必须是一个类类型的对象，右侧运算对象必须是该类型对象的一个成员名（成员函数有时候也被称为方法），运算结果为右侧运算对象指定的成员。
